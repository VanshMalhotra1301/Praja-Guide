How PrajaGuide Works

A deep dive into the intelligent engine powering our citizen-to-scheme matching platform. From raw user data to deterministic output.

Project Overview

1. Profile Ingestion

Users provide their demographic data, socio-economic status, and specific needs through a conversational UI. We collect over 30 distinct data points.

2. AI & Rule Processing

The system passes the profile through a Natural Language Processor, a Bi-LSTM Neural Network, and a deterministic Rule Engine concurrently.

3. Precision Matching

The final output is a highly curated list of government schemes, ranked by an eligibility score, complete with AI-generated explanations.

NLP Data Processing

Before the AI can understand the citizen's profile, the raw input text must be structured and normalized. Here is the exact preprocessing pipeline applied in real-time.

Step 1: Text Sanitization

Lowercasing, regex filtering [^a-z0-9\s] to remove special characters, and collapsing multiple spaces to ensure clean input data.

Step 2: Tokenization

Using a pre-trained Keras Tokenizer (tokenizer.pickle) mapping over 20,000 unique semantic tokens from the Indian government scheme corpus.

Step 3: Sequence Padding

Sequences are standardized to exactly 250 timesteps using pad_sequences(maxlen=250, padding='post', truncating='post') preparing it for the Neural Network.

ML Model Pipeline

Bi-LSTM Engine

Our AI model utilizes a Bidirectional Long Short-Term Memory (Bi-LSTM) architecture to predict missing categories based on user context.

  • 100-Dimensional Vector Space for deep semantic mapping.
  • 128 Recurrent Units capturing forward and backward context.
  • Multi-Label Classification threshold at 0.20 probability.

Result Generation Logic

The ML parameters are passed to a highly complex Deterministic Rule Engine that enforces government guidelines strictly.

Hard Rejection Filters

  • State/Region mismatches
  • Gender-exclusive schemes
  • Caste restrictions (e.g., SC/ST only)
  • Strict Age & Income bounds parsed via Regex
  • Occupation requirements (e.g., Farmer only)

Scoring Algorithm

  • +100 Primary Need Match
  • +50 State Match
  • +40 Occupation Specifics
  • +20-35 Vulnerability Bonus (Widow, Kutcha House, Single Parent)
  • +25 AI Category Match

Final Enrichment

Scores are normalized to a 60-100% scale. The engine generates dynamic, tailored AI Explanations, strictly assigns required Documents, and estimates Financial Benefits via Regex extraction.
